# azure-toolkit

> Operate Azure the way a Cloud Architect does — PaaS-first, identity through Entra ID, least-privilege RBAC, private, cost-aware, against the Well-Architected pillars. Use when provisioning, reviewing, or hardening Azure; picking compute (App Service, Container Apps, Functions, AKS, VM) or data (Azure SQL, Cosmos DB); writing Bicep or Terraform; setting up Entra ID, managed identities, Key Vault, or landing zones; sizing for cost; or mapping to SOC2. IaC-first; a deploy is approval-gated.

## What it does

1. State the workload and its bar.
2. Choose the most managed service that meets the constraint.
3. Apply the five Well-Architected pillars.
4. Lay the identity and governance baseline.
5. Author as IaC and attach cost controls.
6. Review against Well-Architected and the red flags.
7. Gate the mutation on a reviewed plan.
