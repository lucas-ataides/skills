# grill-me

> Extract the knowledge only the user holds through pointed, doc-grounded questions asked one at a time, recorded into a target document until answers saturate. Use when the user wants to be interviewed or grilled, capture or sharpen requirements, fill a spec from a vague feature request, surface hidden assumptions, pin down edge cases and constraints, or turn tacit knowledge into a written artifact. When the problem itself is still unshaped, use brainstorm instead.

**Model-invoked** — the agent runs it automatically when your request matches the triggers below. You can also invoke it by name.

## When to use

- be interviewed

- surface hidden assumptions
- pin down edge cases and constraints
- turn tacit knowledge into a written artifact

## What it does

1. Read what exists, then name the gaps.
2. Name the target document.
3. Ask one grounded question, then wait.
4. Press a vague answer into a concrete one.
5. Record the answer before the next question.
6. Stop at saturation.
