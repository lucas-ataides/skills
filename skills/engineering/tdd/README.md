# tdd

> Build features and fix bugs test-first — red, green, refactor — with the loop enforced by the test gate. Use when the user wants to add a feature, fix a bug, change behavior under tests, or reproduce a defect with a failing test before fixing it.

**Model-invoked** — the agent runs it automatically when your request matches the triggers below. You can also invoke it by name.

## When to use

- add a feature
- fix a bug
- change behavior under tests
- reproduce a defect with a failing test before fixing it

## What it does

1. Frame the next behavior.
2. Red — write one failing test.
3. Green — write the least code that passes.
4. Refactor — improve the design on green.
5. Harden the change.
6. Advance or close.
